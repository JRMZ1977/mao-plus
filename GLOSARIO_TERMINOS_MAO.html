I

Principios del glosario

Este glosario define como deben nombrarse las metricas y conceptos en MAO Plus para mantener coherencia metodologica y evitar confusiones entre metricas 2D, indicadores 3D y terminos de interfaz.

Regla de prioridad

En textos para usuario, mostrar primero el termino canonico en espanol. Si se necesita alias en ingles, incluirlo entre parentesis.

Regla de no mezcla

No usar metricas 3D PCA (linearity, sphericity) como si fueran metricas 2D (elongacion, circularidad).

II

Terminos 2D canonicos

Termino canonico Alias permitido Evitar como etiqueta principal Definicion operativa
Relacion de aspecto AR Aspect ratio Eje mayor / eje menor
Elongacion Elongacion normalizada Elongation (solo EN) Indice de alargamiento normalizado en [0,1] cuando aplica
Circularidad - Circularity como etiqueta unica 4*pi*Area / Perimetro^2
Solidez - Solidity como etiqueta unica Area real / area del envolvente convexo (hull)
Convexidad - Convexity como etiqueta unica Relacion entre contorno convexo y contorno real
III

Perforaciones y horadaciones

Termino Codigo Definicion Evitar
Perforacion P Abertura pasante que conecta caras opuestas Hueco (ambiguo)
Horadacion H Depresion no pasante o cavidad superficial Hueco (ambiguo)
Cuando no se conozca el tipo de vacio aun, usar "vacio" solo de forma temporal en diagnostico interno, nunca como etiqueta final en reportes.
IV

Terminos 3D y PCA

Termino 3D Uso correcto No equivalente automatico con
Linealidad (PCA) Describe dominancia del eje principal en nube 3D Elongacion 2D
Esfericidad (PCA/3D) Describe proximidad volumetrica a forma esferica Circularidad 2D
Planaridad Describe tendencia de puntos a un plano Rectangularidad 2D
Anisotropia Contraste direccional de forma 3D Relacion de aspecto 2D
V

Bifacialidad y caras

Termino canonico Alias Regla de presentacion
Anverso / Reverso FRONT / BACK En UI final: "Anverso/Reverso (FRONT/BACK)" cuando haya modo bilingue
Balance bifacial Bifacial balance Reportar con componentes (area, perimetro, forma)
Homologia bifacial Bifacial homology index No confundir con CI/CMS si el modo no lo activa
Para consistencia metodologica, usar siempre la misma pareja semantica de caras en toda la vista: no mezclar "Cara A/B" con "Front/Back" sin equivalencia explicita en la misma pantalla.
VI

Reglas de uso rapido

Checklist para UI y reportes

  • Usar "Relacion de aspecto (AR)" como etiqueta principal.
  • Mantener "Elongacion" para indice normalizado, no para linealidad PCA.
  • Mostrar "Anverso/Reverso (FRONT/BACK)" en contexto bilingue.
  • Diferenciar siempre Perforacion (P) y Horadacion (H).
  • No usar terminos ambiguos como etiqueta final (ejemplo: "hueco").

Aplicacion recomendada

Este glosario debe consultarse antes de editar archivos de interfaz, exportacion CSV/JSON y documentacion tecnica. Se recomienda versionar cambios semanticos con nota de migracion.
